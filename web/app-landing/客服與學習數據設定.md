# 官網客服寄信與學生學習數據設定

## 一、免費客服表單（Web3Forms，不需自建 SMTP）

1. 開啟 https://web3forms.com 註冊（免費方案每月約 250 封）。
2. 建立表單，收件 Email 設為 **sgqaiapp@gmail.com**。
3. 複製 **Access Key**，貼到 `config.js`：

```javascript
web3formsAccessKey: '你的-access-key',
```

4. 部署到 EC2（`run-app-domain.bat`）。
5. 在官網「聯絡客服」送出一則測試訊息，確認信箱有收到。

`web3formsAccessKey` 留空時，表單仍會用 **mailto**（開啟郵件 App）。

### Google reCAPTCHA（目前暫停）

官網已暫時移除 reCAPTCHA。若要恢復：還原 `index.html` / `student.html` 的驗證區塊、載入 `sgq-recaptcha.js`，並在 `config.js` 填入 `recaptchaVersion` 與 `recaptchaSiteKey`。

### Google reCAPTCHA（客服表單與學生登入，恢復時參考）

1. 至 https://www.google.com/recaptcha/admin 建立 **reCAPTCHA v2** →「我不是機器人」核取方塊。
2. 網域加入：`app.sagp-qp.com`（本機測試可加 `localhost`）。
3. 將 **Site Key** 貼到 `config.js` 的 `recaptchaSiteKey`。
4. 若使用 Web3Forms：至 Web3Forms 後台表單設定 → Captcha → 選 reCAPTCHA，貼上 **Secret Key** 並儲存。
5. `config.js` 設 `recaptchaVersion: 'v2'`。重新部署後會顯示「我不是機器人」核取方塊。

`recaptchaSiteKey` 留空時，表單無法通過驗證（會提示尚未設定）。

### 其他免費方案（擇一）

| 服務 | 說明 |
|------|------|
| Web3Forms | 已整合，設定最簡單 |
| Formspree | 類似，需改 `app.js` 端點 |
| EmailJS | 可連 Gmail，設定較繁 |

不建議在靜態網站直接寫 Gmail SMTP 密碼（會外洩）。

## 二、學生學習數據頁

網址：**https://app.sagp-qp.com/student.html**

- 學生用 **與 App 相同** Email / 密碼登入。
- 顯示：題目數、徽章、學習次數、各階段題數、最近題目、各課程統計。
- 教師帳號會被拒絕（請用 export 站）。
- 資料權限由 Supabase **RLS** 保護，只能看自己的資料。

### Supabase 必設（與 export 站相同）

**Authentication → URL Configuration**

| 欄位 | 填入 |
|------|------|
| Site URL | `https://app.sagp-qp.com` |
| Redirect URLs | `https://app.sagp-qp.com`、`https://app.sagp-qp.com/student.html` |

本機測試可加：`http://localhost:8081`

---

## 三、部署

修改 `config.js`、`student.html` 等後執行 **`run-app-domain.bat`**。
